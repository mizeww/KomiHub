# KomiHub
Проект представляет из себя веб-приложение, которое направлено на помощь в изучении коми языка. Проект содержит карточки с заданиями, тренажеры и конспект по коми языку. Это будет давать необходимую теоретическую и практическую базу для изучения языка.
У  каждого пользователя есть возможность создать аккаунт для контроля и отслеживания личной статистики и соревнования с другими.
При входе на сайт пользователь будет обязан полностью прослушать гимн Республики Коми. 

1. Введение и цели
Цель проекта: Разработать веб-приложение для изучения коми языка, предоставляющее пользователям теоретическую базу (конспекты) и практические инструменты (карточки, тренажеры) для формирования устойчивых языковых навыков.

Целевая аудитория: Энтузиасты, студенты, а также все желающие познакомиться с культурой и языком Республики Коми.

Ключевая особенность (User Flow): При первом входе на сайт пользователь обязан полностью прослушать Гимн Республики Коми, после чего получает доступ к функционалу.

2. Функциональные требования (План действий)
2.2. Модуль пользователей (Аккаунты и статистика)
Регистрация / Авторизация:

Стандартная форма (логин/пароль) с хешированием пароля.

Валидация полей и проверка уникальности пользователя.

Личный кабинет:

Отображение личной статистики (всего выучено слов, пройдено уроков, текущий "стрик" — серия дней занятий).

Возможность изменения аватара (загрузка картинки).

Таблица лидеров: Страница с рейтингом пользователей на основе количества набранных очков за упражнения (соревновательный момент).

2.3. Модуль конспектов (Теория)
База данных (БД): Таблицы с разделами, темами и статьями.

Получение данных: Динамическая подгрузка содержимого конспекта при клике на тему (без перезагрузки всей страницы).

Структурирование: Возможность добавлять в конспекты таблицы, примеры и выделение грамматических правил.

2.4. Модуль карточек (Запоминание слов)
CUP (Cards/User/Progress): Интерактивные карточки "Слово на коми / Перевод на русский".

Алгоритм показа: Показ карточек, где пользователь оценивает, насколько легко он запомнил слово ("Легко", "Тяжело", "Повторить").

Хранение прогресса: База данных фиксирует, когда пользователь в последний раз видел карточку, и при следующем сеансе показывает сложные слова чаще.

2.5. Модуль тренажеров (Практика)
Вариативность заданий: (Например: "Вставь пропущенную букву", "Составь предложение", "Выбери верный перевод").

Генерация контента: Приложение выбирает вопросы из базы данных на основе текущего уровня пользователя.

Проверка результатов: Мгновенная обратная связь "Верно/Неверно" с начислением очков.

2.6. Администрирование и данные
Панель администратора: Возможность добавления новых слов в базу, редактирования конспектов без прямого доступа к коду (через интерфейс).

Логирование: Запись действий пользователя (вход, выход, выполнение упражнений) в текстовые логи.

3. Технические требования
Тип приложения: Веб-приложение.

База данных: SQL.

Хранение: На сервере хранятся: аватары пользователей, файлы логов, аудиофайл гимна.

Выгрузка данных: Функция экспорта прогресса пользователя в базу данных.

4. Сценарии взаимодействия (Пример)
Пользователь заходит на сайт -> Видит приветствие и включает Гимн -> Кнопка "Начать обучение" разблокируется.

Пользователь регистрируется -> Данные сохраняются в БД.

Пользователь открывает раздел "Тренажер" -> Приложение выбирает 10 случайных слов из БД по его уровню. Пользователь нажимает кнопки, приложение воспроизводит аудио-клик (звук нажатия/правильного ответа).

Пользователь выгружает отчет -> Система формирует его статистику по дням.

5. План реализации (Дорожная карта)
Проектирование БД: Таблицы users, progress, words, и тд.

Backend: Реализация API для регистрации, авторизации, получения карточек и конспектов.

Frontend: Верстка плеера с гимном, сетка конспектов, виджет карточек.
